1) 한양대학교 취업상담 포털 (CRM+LMS 통합 대시보드)
900명+ 학생·상담사·코디네이터·관리자 4개 역할별 대시보드를 구축했습니다. 코디네이터 대시보드에서 상담 현황 모니터링, 학생 프로필 관리, 월별 상담유형 통계, 설문 응답 분석 등 CRM 핵심 기능을 구현했고, WebSocket 기반 실시간 알림과 FullCalendar 예약 시스템을 연동했습니다.

2) 심다(PlantBid) 판매자 관리 대시보드
입찰·주문·정산·고객 커뮤니케이션을 통합 관리하는 B2B 어드민 대시보드를 구축했습니다. 주문 상태 트래킹, 실시간 채팅, 매출 분석, 입찰 전환율 등 핵심 지표를 한 화면에서 확인할 수 있도록 설계했으며, 사업자 인증(국세청 API)과 PortOne V2 정산 시스템까지 연동했습니다. 두 프로젝트 모두 React + PostgreSQL + Drizzle ORM 기반으로, 본 프로젝트와 동일한 데이터 정합성 중심의 내부 관리 도구입니다.

## 프로젝트 진행 제안

Studio HaeTae | 12년 대기업 연구원 출신, 현재 스타트업 CTO 재직중

본 프로젝트를 3단계 스프린트로 제안드립니다.

**1단계 (1~2주): 설계 및 기반 구축**
- 기부자 프로필 정의서 기반 DB 스키마 설계
- 내부 시스템 API 연동 명세 확인 및 테스트 데이터 검증
- 뉴스 수집 소스 선정 (네이버 뉴스 API, DART 공시 API 등)
- 화면 설계서 및 기능 명세서 산출

**2단계 (3~5주): 핵심 기능 개발**
- 900명 기부자 명단 기반 자동 뉴스 수집 파이프라인 구축 (1일 1회 정기 수집)
- LLM(Claude API) 기반 동명이인 판별 + 기사 핵심 요약 기능
- 기부 이력 데이터와 뉴스 데이터 병합 및 골든타임 점수화 로직 설계
- 관리자 대시보드 프론트엔드 개발 (기부자 목록, 뉴스 타임라인, 점수 랭킹)

**3단계 (6~7주): 안정화 및 납품**
- 사내 10명 대상 UAT(사용자 수용 테스트)
- 데이터 정합성 검증 및 엣지케이스 처리
- 시스템 아키텍처 설계도 + 원본 소스코드 + 기능 명세서 납품
- 운영 가이드 및 LLM API 월 비용 리포트 제공

**월 유지 비용 예상**: LLM API(Claude) 약 3~8만원/월 + Supabase Pro 약 2.5만원/월 = 월 5~10만원 수준으로, 900명 기준 일 1회 수집 시 충분히 경제적인 운영이 가능합니다.

1차 예산(600만원) 내에서 핵심 기능을 안정적으로 구현하고, 추후 고도화(SNS 연동, 점수 모델 고도화, 알림 시스템 등)를 단계적으로 진행하는 구조로 제안드립니다.

## 관련 포트폴리오 설명

**1. 한양대학교 취업상담 포털 (CRM + LMS 통합 대시보드)**

대학교 취업지원센터의 상담 업무를 디지털화한 풀스택 프로젝트입니다. 학생 900명+, 상담사, 코디네이터, 관리자 4개 역할별로 완전히 다른 대시보드를 설계·구현했습니다.

핵심 과제는 본 프로젝트와 매우 유사했습니다. 파편화된 학생 데이터(상담 이력, 설문 응답, 예약 현황)를 한 곳에 모아 코디네이터가 한눈에 파악할 수 있는 통합 모니터링 환경을 만드는 것이었습니다. 코디네이터 대시보드에서 월별 상담유형 통계, 상담사별 실적, 설문 응답 분석 등 다차원 데이터를 시각화했고, WebSocket 기반 실시간 알림으로 신규 예약이나 취소를 즉시 감지하도록 구현했습니다. FullCalendar 연동 예약 시스템, Zoom 화상상담, Vimeo LMS까지 통합하여 하나의 플랫폼에서 모든 업무가 처리되는 환경을 구축했습니다.

특히 동시 예약 충돌 방지를 위해 PostgreSQL 트랜잭션 기반 동시성 제어를 적용했고, 48시간 전 취소 규칙 등 복잡한 비즈니스 로직을 DB 레벨에서 처리하여 데이터 정합성을 확보한 경험이 있습니다. 사내 10명+ 직원이 상시 사용하는 B2B 내부 도구라는 점에서 본 프로젝트와 운영 환경이 동일합니다.
기술스택: React, Express, PostgreSQL, Drizzle ORM, WebSocket, Firebase

**2. 심다(PlantBid) 판매자 관리 대시보드**

AI 식물 추천 엔진과 위치 기반 입찰을 결합한 O2O 커머스 플랫폼에서 판매자용 CRM 대시보드를 구축했습니다.

판매자가 하나의 화면에서 주문 관리(상태 트래킹, 배송/픽업 정보, 결제 내역), 입찰 관리(실시간 타이머, 2시간 마감 카운트다운, 수락/거절), 고객 커뮤니케이션(1:1 채팅, 주문별 대화), 매출 분석(판매 실적, 입찰 전환율, 주문 이력)을 통합 관리합니다. 국세청 API를 연동한 사업자 인증, PortOne V2 정산 시스템, 은행 계좌 검증까지 외부 데이터 연동이 핵심이었습니다.

본 프로젝트와의 유사점은 "외부 데이터(국세청, 결제사, 지도 API)를 수집하여 내부 판매자 데이터와 병합하고, 의사결정에 필요한 핵심 지표를 대시보드로 시각화"하는 구조입니다. Gemini AI를 활용한 상품 추천 로직도 구현하여 AI API 연동 경험을 보유하고 있습니다.
기술스택: React, Express, PostgreSQL, Drizzle ORM, Gemini AI, Google Maps API, PortOne V2

**3. 퍼포먼스 마케팅 SaaS (외부 API 자동 수집 + AI 분석 대시보드)**

YouTube, Instagram, TikTok 공식 API를 연동하여 크리에이터의 콘텐츠 퍼포먼스를 실시간 수집하고, Gemini AI로 콘텐츠를 자동 분석하는 B2B SaaS 대시보드입니다.

이 프로젝트의 핵심 파이프라인이 본 프로젝트와 구조적으로 거의 동일합니다. (1) 외부 API에서 데이터를 주기적으로 자동 수집 → (2) AI가 수집된 데이터를 분석·요약 → (3) 내부 데이터와 병합하여 대시보드에 시각화. 본 프로젝트에서는 이 파이프라인이 "뉴스 API로 기사 수집 → Claude가 동명이인 판별·요약 → 기부 이력과 병합하여 골든타임 점수화"로 적용됩니다.

특히 SNS API의 Rate Limit 관리, 대량 데이터 배치 처리, 수집 실패 시 재시도 로직 등 외부 데이터 수집 파이프라인의 안정성 확보 경험이 있어, 900명 기부자의 일일 뉴스 수집에도 안정적인 운영이 가능합니다. PortOne 파트너 정산 자동화 시스템도 연동하여 복합 B2B 시스템 구축 경험을 보유하고 있습니다.
기술스택: React, Express, Drizzle ORM, Gemini AI, YouTube/Instagram/TikTok API, PortOne

## 사용 기술과 툴

**프론트엔드**: Next.js + TypeScript + Tailwind CSS
→ 사내 10명이 사용하는 관리 도구에 최적. SSR로 초기 로딩 빠르고, CRM 대시보드 구축 경험이 있는 검증된 스택.

**백엔드/DB/인프라**: Supabase (PostgreSQL + Edge Functions + Auth)
→ 별도 서버 구축 없이 DB·API·인증·스케줄링을 통합 제공. 기부자 900명 + 사내 10명 규모에 최적이며, 월 유지비를 서버 0원(Free) ~ 2.5만원(Pro)으로 최소화할 수 있습니다. 향후 확장 시에도 PostgreSQL 기반이라 마이그레이션 부담 없음.

**뉴스 수집**: 네이버 뉴스 검색 API + DART 공시 API (1차)
→ Supabase Edge Function에서 API 호출로 경제적이고 안정적인 수집. pg_cron으로 매일 자동 스케줄링. 추후 고도화 시 BeautifulSoup/Selenium 기반 웹 크롤링으로 언론사 직접 수집, SNS(Meta Graph API) 모니터링 등 확장 가능합니다.

**AI 분석**: Claude API (Anthropic)
→ 동명이인 판별과 기사 요약에 가장 정확한 한국어 LLM. 900명 × 일 1회 기준 월 3~8만원.

**월 총 유지비용: 약 5~10만원** (LLM API + Supabase Pro)
