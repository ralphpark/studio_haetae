## 클라이언트 질문 답변

### Q1. 교육 플랫폼 유사 개발 경험 있으신가요? (500자)

한양대학교 취업상담 포털을 풀스택으로 개발한 경험이 있습니다. 900명+ 학생이 사용하는 교육·상담 통합 플랫폼으로, 본 프로젝트와 유사하게 학생·교육자(상담사)·관리자(코디네이터) 역할별 대시보드를 구축했습니다. Vimeo 연동 LMS에서 온라인 교육 수강, 진도율 추적, 수료 판정 기능을 구현했고, FullCalendar 기반 실시간 예약 시스템과 WebSocket 실시간 알림을 연동했습니다. 관리자 대시보드에서 학생별 이수 현황 조회, 월별 통계 시각화, 설문 응답 분석 등 본 프로젝트의 교사 대시보드(학생 제출 현황, 답변 조회)와 동일한 패턴의 기능을 구현했습니다. React + Express + PostgreSQL + Drizzle ORM 기반이며, 역할별 접근 제어와 데이터 정합성이 핵심이었습니다.

### Q2. Next.js 14 App Router 실무 경험 있으신가요? (500자)

현재 운영 중인 에이전시 웹사이트(haetae.studio)를 Next.js App Router + TypeScript + Tailwind CSS로 개발하여 Vercel에 배포·운영하고 있습니다. Server Components와 Client Components 분리, generateMetadata를 활용한 동적 메타태그 생성, Route Handlers로 API 구현, middleware(proxy.ts)를 통한 인증 가드 등 App Router의 핵심 패턴을 프로덕션 레벨에서 활용하고 있습니다. Supabase Auth 연동, sessionStorage 기반 폼 데이터 임시 저장 → 인증 후 자동 전송 플로우, Framer Motion 애니메이션, Lenis 스무스 스크롤 등을 구현했으며, 현재 Next.js 16까지 마이그레이션하여 운영 중입니다. Vercel 배포 환경에서 ISR, Edge Runtime, 환경변수 관리 등 실무 경험을 보유하고 있습니다.

### Q3. Supabase RLS 설계 경험 있으신가요? (500자)

haetae.studio 프로젝트에서 Supabase RLS(Row Level Security)를 설계·적용하여 운영 중입니다. 상담 문의 테이블에서 "본인이 작성한 문의만 조회 가능", 관리자 역할은 전체 조회 가능, 비인증 사용자는 INSERT만 가능(조회 불가) 등 역할 기반 정책을 구현했습니다. 또한 Supabase Auth의 auth.uid()와 JWT claims를 활용하여 사용자별 데이터 격리를 DB 레벨에서 보장합니다. 본 프로젝트에 적용할 RLS 설계안으로는, 학생은 자신의 워크북 답변만 CRUD 가능, 교사는 자신의 클래스에 속한 학생 답변만 조회 가능, 관리자는 학교 단위 전체 데이터 접근 가능으로 설계할 수 있습니다. 클라이언트가 제공하는 DB 스키마 SQL에 맞춰 RLS 정책을 즉시 적용할 수 있습니다.

## 프로젝트 진행 제안

Studio HaeTae | 12년 대기업 연구원 출신, 현재 스타트업 CTO 재직중

클라이언트가 DB 스키마, 기능 스펙, UI 프로토타입을 이미 준비하셨으므로 바로 개발에 착수 가능합니다. 5주 일정에 맞춰 진행합니다.

**1주차: 기반 구축 + 인프라**
- 제공된 DB 스키마 SQL 적용 및 RLS 정책 설정
- Supabase 프로젝트 세팅 (Auth, Storage, Realtime)
- Next.js 14 프로젝트 초기 구성 + Vercel 배포 파이프라인
- UI 프로토타입 HTML 4종 → Next.js 컴포넌트 변환 착수

**2주차: 학생 기능**
- 클래스 코드 입장 플로우 개발
- 워크북 작성 UI (스마트기기 최적화, 터치 인터랙션)
- 자동 저장 (디바운스 + Supabase Realtime) 및 최종 제출 기능

**3주차: 교사 기능**
- 실시간 학생 제출 현황 대시보드 (Supabase Realtime 구독)
- 학생별 답변 조회 기능
- 전자칠판용 수업 화면 (질문 표시, 학생 답변 공개, 포커스 모드)

**4주차: 관리자 + 연동**

- 전자칠판 ↔ 학생 기기 실시간 동기화 안정화

**5주차: QA + 납품**
- 크로스 디바이스 테스트 (PC, 태블릿, 모바일)
- 동시 접속 부하 테스트 (한 클래스 30명 동시 제출)
- 엣지케이스 처리 및 버그 수정
- 소스 코드 납품

## 관련 포트폴리오 설명 (2000자)

**1. 한양대학교 취업상담 포털 (교육 플랫폼 + LMS + 역할별 대시보드)**

본 프로젝트와 가장 직접적으로 연결되는 교육 플랫폼 개발 경험입니다. 대학교 취업지원센터의 상담·교육 업무를 디지털화한 풀스택 시스템으로, 학생 900명+, 상담사, 코디네이터, 관리자 4개 역할별로 완전히 다른 대시보드를 설계·구현했습니다.

본 프로젝트의 "학생이 워크북을 작성하고 교사가 실시간으로 확인"하는 구조와 동일하게, 학생이 상담을 신청하면 코디네이터 대시보드에 실시간으로 반영되고, 코디네이터가 승인/반려 처리를 하면 학생에게 즉시 알림이 발송되는 양방향 실시간 인터랙션을 구현했습니다. WebSocket 기반 실시간 알림으로 신규 예약, 상태 변경, 취소 등 모든 이벤트를 즉시 전파합니다.

LMS 기능으로 Vimeo API를 연동하여 온라인 교육 영상 시청, 진도율 자동 추적, 교육 이수 판정을 구현했습니다. 관리자 대시보드에서는 학생별 이수 현황, 월별 상담유형 통계, 상담사별 실적, 설문 응답 분석 등 다차원 데이터를 시각화합니다. 이 패턴은 본 프로젝트의 교사 대시보드(학생별 제출 현황, 답변 조회)에 그대로 적용됩니다.

동시 예약 충돌 방지를 위해 PostgreSQL 트랜잭션 기반 동시성 제어를 적용했으며, 이는 본 프로젝트에서 30명 학생이 동시에 워크북을 제출할 때의 데이터 정합성 보장에 동일하게 활용할 수 있습니다. React + Express + PostgreSQL + Drizzle ORM + WebSocket + Firebase 기반.

**2. Studio HaeTae (haetae.studio) - Next.js App Router + Supabase + Vercel**

본 프로젝트와 완전히 동일한 기술 스택으로 개발·운영 중인 프로덕션 서비스입니다. Next.js App Router의 Server Components, Client Components, Route Handlers, Middleware, generateMetadata 등 핵심 패턴을 모두 실무에서 활용하고 있습니다.

Supabase Auth 기반 인증 시스템(로그인/회원가입/비밀번호 찾기/로그아웃)을 구현했으며, middleware에서 미인증 사용자를 로그인 페이지로 리다이렉트하는 인증 가드를 적용했습니다. 이 패턴을 본 프로젝트에서는 학생은 클래스 코드로 입장, 교사는 로그인 후 대시보드 접근, 관리자는 관리자 페이지 접근으로 변환하여 적용합니다.

Supabase RLS를 실무에서 설계·운영한 경험이 있어, 클라이언트가 제공하는 DB 스키마 SQL에 맞춰 학생/교사/관리자 역할별 데이터 접근 정책을 즉시 적용할 수 있습니다. Vercel 배포 환경에서 환경변수 관리, Preview Deployments를 활용한 PR별 테스트, 프로덕션 무중단 배포까지 실무 경험을 보유하고 있습니다.

**3. SNS 콘텐츠 자동화 시스템 (실시간 데이터 파이프라인)**

n8n + Supabase Realtime을 활용하여 콘텐츠 생성 → 스케줄링 → 자동 발행 → 결과 리포팅까지의 실시간 데이터 파이프라인을 구축·운영 중입니다. Supabase Realtime의 Broadcast와 Presence 채널을 실무에서 활용한 경험이 있어, 본 프로젝트의 핵심 기능인 전자칠판 ↔ 학생 기기 간 실시간 동기화(질문 표시, 답변 공개, 포커스 모드)를 안정적으로 구현할 수 있습니다. 특히 하루 3회 자동 발행 스케줄러에서 Supabase DB 상태 변경 → Realtime 구독 → 즉시 UI 반영 패턴을 검증했으며, 이는 학생 워크북 제출 → 교사 대시보드 실시간 업데이트와 동일한 구조입니다.
